Sobre esta Aplicação

1. Arquitetura MVT do Django

A aplicação foi desenvolvida com base na arquitetura MVT (Model-View-Template).

Componentes:

  • Model: responsável pelos dados e pela base de dados.
  • View: responsável pela lógica da aplicação.
  • Template: responsável pela interface gráfica.

Fluxo de informação:

O utilizador faz um pedido → a View processa → consulta o Model → envia os dados para o Template → o Template apresenta ao utilizador.

2. Modelação

O sistema foi modelado com base num Diagrama Entidade-Relação.

Exemplo de models.py:

class Projeto(models.Model):
    nome = models.CharField(max_length=100)
    descricao = models.TextField()
    unidade_curricular = models.ForeignKey(UnidadeCurricular, on_delete=models.CASCADE)
    tecnologias = models.ManyToManyField(Tecnologia)
    competencias = models.ManyToManyField(Competencia)
